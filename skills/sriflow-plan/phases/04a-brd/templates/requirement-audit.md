# Requirement Audit Template

## Uncertainty Reduction Scale

| Score | Build-readiness |
|-------|----------------|
| 0-2 | Do not build |
| 3-4 | Do not build |
| 5-6 | Only if gaps are documented risk |
| 7-8 | Acceptable for sprint |
| 9-10 | Ready |

---

## Requirement Checklist

For each requirement, verify:

| # | Check | Pass/Fail | Notes |
|---|-------|-----------|-------|
| 1 | Actor named? (no "users" without role) | | |
| 2 | Verb-Noun structure? (action on a thing) | | |
| 3 | Trigger specified? (what event activates) | | |
| 4 | Scope bounded? (no "etc.") | | |
| 5 | Measurable? (number, time, percentage) | | |
| 6 | Traceable? (links to a UC) | | |
| 7 | Testable? (QA can write pass/fail) | | |
